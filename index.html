Module 1: Introduction

What is HTML & Evolution

HyperText Markup Language

HTML is the structural foundation of the Web. It defines the layout, sections, links, and content containers of a page. HTML uses tags to define how text, images, and other resources are rendered in the browser.

  • HTML: Flexible standard framework for pages.
  • XHTML: Stricter, XML-validated version of HTML.
  • HTML5: Modern specification offering native media assets.

HTML 1.0 - The Genesis (1991)

Created by Tim Berners-Lee at CERN. It contained only 18 basic structural elements like paragraphs, headings, lists, and anchors. Design layout controls, colors, or external stylesheets did not exist yet.

Module 3: Structuring Lists

HTML Lists Layouts

Ordered Lists (`<ol>`)

Used to display items in structured numerical or alphabetical sequences.

  • type: Defines numbering format ('1', 'A', 'a', 'I', 'i').
  • start: Starting numeric index offset (e.g. `start="5"`).
  • reversed: Boolean property to count backwards.
HTML Code 
<ol type="A" start="2" reversed>
  <li>Step Two (B)</li>
  <li>Step One (A)</li>
</ol>

Unordered Lists (`<ul>`)

Displays items in bulleted layouts without designated numerical order.

  • type: Defines bullet shapes: 'disc' (default fill), 'circle' (hollow), or 'square'.
  • Note: In modern development, bullet shapes are customized using CSS `list-style-type` styling.
HTML Code 
<ul type="circle">
  <li>Red Apple</li>
  <li>Yellow Mango</li>
</ul>

Description Lists (`<dl>`)

Creates terminological glossaries consisting of terms paired directly with description blocks.

  • <dt>: Description Term, representing the defined word.
  • <dd>: Description Detail, wrapping the definition content block.
HTML Code 
<dl>
  <dt>HTML</dt>
  <dd>HyperText Markup Language</dd>
  <dt>CSS</dt>
  <dd>Cascading Style Sheets</dd>
</dl>

Module 4: Rich Media

Images, Video & Audio Embeds

🖼️ Images (<img>)

Embeds graphic items. It is a self-closing structural element.

<img src="flower.jpg" 
     alt="Flower" 
     width="300" 
     height="200" 
     title="Hover tip">
  • src: File source path or address.
  • alt: Accessible text fallbacks.

🎬 Videos (<video>)

Embeds media clips. Supports playback parameters.

<video src="movie.mp4" 
       controls 
       autoplay 
       muted 
       loop 
       poster="thumb.jpg">
</video>
  • controls: Enables player sliders.
  • muted: Plays audio muted first.

🎵 Audio (<audio>)

Embeds music elements using multiple sources.

<audio controls loop>
  <source src="song.mp3" 
          type="audio/mpeg">
  <source src="song.ogg" 
          type="audio/ogg">
</audio>
  • <source>: Alternative formats fallback support list.

Module 5: Forms

HTML Forms & Input Forms

Collecting Input Values

Forms gather user inputs using diverse tags enclosed in a parent <form> container tag.

  • action: Target backend endpoint URL handler to submit data.
  • method: Data submission process: get (in URL) or post (secure request payload).
  • <input> types: text, password, email, radio, checkbox, date, file, color.
  • <select> & <option>: Renders select dropdowns.

Form Layout Syntax

HTML Form 
<form action="/submit" method="post">
  <label for="name">Name:</label>
  <input type="text" id="name" required>

  <label for="city">City:</label>
  <select id="city">
    <option value="KHI">Karachi</option>
    <option value="LHR">Lahore</option>
  </select>

  <button type="submit">Register</button>
</form>
Module 5: Architecture

HTML5 Semantic Layout Architecture

Meaningful Tag Layouts

Semantic elements communicate meaning to browser agents, search engine scrapers (SEO), and screen reader accessibility tools.

  • <header> / <footer>: Page headers and footers.
  • <nav>: Wraps site navigation anchors.
  • <main>: Wraps the unique primary content.
  • <section> / <article>: Groups themes / standalone articles.
  • <aside>: Sidebar navigation or additional notes.
  • <details> & <summary>: Creates native interactive toggles.

Semantic vs Non-Semantic Structuring

Clean Semantic Layout (Best Practice) 
<header>
  <nav>Menu Links...</nav>
</header>
<main>
  <section>
    <article>Blog Post Content</article>
  </section>
</main>
<footer>Copyright © 2025 Shayan</footer>
Module 6: Entities

HTML Entity Special Characters

Displaying Escaped Symbols

Special symbols or characters that conflict with HTML tag syntax (like < or >) must be escaped using HTML entities starting with & and ending with ;.

© &copy; Copyright
® &reg; Registered
™ &trade; Trademark
€ &euro; Euro Sign
& &amp; Ampersand
< &lt; Less Than
> &gt; Greater Than
